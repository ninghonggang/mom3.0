# MOM3.0 vs 峰梅动力需求 差距分析

## 项目背景
- **甲方**: 上海峰梅动力系统有限公司
- **项目**: MES系统建设
- **上线要求**: 2026年7月15日前
- **核心工艺**: 发动机部件机加工、磨削、焊接、铆接、涂胶、清洗、装配、气密、装箱
- **三个车间**: 平衡轴车间(2条线)、电池包车间(2条线)、试制车间(≤10台设备)

## 六、关键差距总结

### 核心生产执行闭环 (缺失严重)
```
需求: 原材料 → 上料(AGV+机器人) → 加工 → 焊接 → 检测(视觉) → 打印条码 → 装箱 → 出库
当前: 工单/派工/报工基础完成
缺失: 原材料追溯, 视觉检测对接, 箱条码, AGV, JIT拉动, 精确批次
```

### 质量管理 (几乎空白)
```
需求: IQC → 过程检(IPQC) → 首末检 → FQC → OQC + SPC + 实验室 + QRCI
当前: IQC前端存在, 后端未完成
缺失: IPQC, 首末检, SPC, 实验室, QRCI, 完整质量闭环
```

### 物料拉动 (空白)
```
需求: JIT/JIS拉动 → 电子看板 → 紧急拉动 → AGV配送
当前: 无
缺失: 完整的物料拉动体系
```

### 数据采集 (空白)
```
需求: 设备状态 + 工艺参数 + 报警 + 视觉检测 + 蓝牙量具
当前: 无
缺失: 数采平台接口, 实时数据处理
```

## 九、需求文档中的特殊要求 (需注意)

1. **源码开放**: 知识产权归双方共有, 甲方有权进行二次开发
2. **平台要求**: SOA架构, B/S框架, 支持Linux/Windows
3. **数据永久保存**: 基础数据永久, 业务数据12个月, 历史数据10年
4. **响应时间**: 事务≤3秒, 查询≤15秒
5. **可用性**: ≥99.9%, 7*24小时
6. **备份**: 实时数据到历史数据库每日归档
7. **多车间**: 未来支持多工厂复制部署
