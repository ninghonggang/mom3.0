**版本**: V2.0 | **所属模块**: M09安灯系统 | **基于**: [MOM3.0_主设计文档](./MOM3.0_主设计文档.md)

---

## 1. 模块概述

### 1.1 功能定位

安灯系统(Andon)实现车间异常状态的实时呼叫、响应、处理和升级闭环，是制造执行系统的核心模块。

### 1.2 核心功能

| 功能 | 说明 |
|------|------|
| Andon呼叫 | 工位端发起异常呼叫 |
| 呼叫响应 | 响应并开始处理 |
| 升级机制 | 超时自动多级升级 |
| 车间广播 | 网络音柱语音播放 |
| 消息推送 | 飞书/企微通知 |
| 统计分析 | 响应时间/频次统计 |

## 3. UI设计规范

### 3.1 异常类型

| 异常类型 | 代码 | 升级超时(默认) |
|----------|------|---------------|
| 设备故障 | EQUIPMENT | 5分钟 |
| 物料短缺 | MATERIAL | 10分钟 |
| 质量问题 | QUALITY | 5分钟 |
| 技术支持 | TECHNICAL | 15分钟 |
| 夹具模具 | TOOLING | 10分钟 |
| 安全环境 | SAFETY | 0分钟(立即升级) |

### 3.2 状态映射

**呼叫状态**

| 状态值 | 标签类型 | 显示文本 |
|--------|----------|----------|
| CALLING | danger | 呼叫中 |
| RESPONDED | primary | 已响应 |
| HANDLING | warning | 处理中 |
| RESOLVED | success | 已解决 |
| CLOSED | info | 已关闭 |

**呼叫等级**

| 等级 | 颜色 | 说明 |
|------|------|------|
| L1 | 蓝色 | 一级响应 |
| L2 | 橙色 | 二级升级 |
| L3 | 红色 | 三级升级 |
| L4 | 深红 | 最高级升级 |

## 4. 业务流程

### 4.1 呼叫响应流程

```
操作工发起呼叫 → 推送通知到班组长 → 大屏灯色变化 → 30秒内响应 → 开始处理 → 完成关闭
```

### 4.2 自动升级流程

```
L1呼叫 → 5分钟超时 → 升级L2(班组长+车间主任) → 5分钟超时 → 升级L3(+语音广播) → 10分钟超时 → 升级L4(+短信)
```

## 5. 数据模型

### 5.1 Andon呼叫记录

| 字段 | 类型 | 说明 |
|------|------|------|
| call_no | VARCHAR(50) | 呼叫编号 |
| workshop_id | BIGINT | 车间ID |
| workstation_id | BIGINT | 工位ID |
| andon_type | VARCHAR(30) | 异常类型 |
| priority | INTEGER | 优先级1-10 |
| description | VARCHAR(500) | 异常描述 |
| call_by | VARCHAR(50) | 呼叫人 |
| call_time | TIMESTAMP | 呼叫时间 |
| response_by | VARCHAR(50) | 响应人 |
| response_time | TIMESTAMP | 响应时间 |
| handle_time | TIMESTAMP | 处理完成时间 |
| status | VARCHAR(20) | 状态 |
| call_level | INTEGER | 当前等级 |
| is_escalated | SMALLINT | 是否已升级 |

### 5.2 升级规则

| 字段 | 类型 | 说明 |
|------|------|------|
| rule_code | VARCHAR(50) | 规则编码 |
| andon_type | VARCHAR(30) | 适用类型 |
| level1_timeout | INTEGER | L1超时(分钟) |
| level2_timeout | INTEGER | L2超时(分钟) |
| level3_timeout | INTEGER | L3超时(分钟) |
| level1_notify_json | JSONB | L1通知对象 |
| audio_enabled | SMALLINT | 启用语音广播 |

## 6. API接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /andon/calls | 发起呼叫 |
| PUT | /andon/calls/:id/respond | 响应呼叫 |
| PUT | /andon/calls/:id/resolve | 处理完成 |
| PUT | /andon/calls/:id/escalate | 手动升级 |
| GET | /andon/calls | 呼叫列表 |
| GET | /andon/statistics | 统计分析 |
| GET | /andon/escalation-rules | 升级规则列表 |
| POST | /andon/escalation-rules | 创建规则 |
| PUT | /andon/escalation-rules/:id | 更新规则 |

## 7. WebSocket实时推送

```
连接: ws://host/api/v1/ws?token=xxx
订阅: {"action": "subscribe", "channel": "andon", "workshop_id": 1}

推送消息:
- ANDON_CALL: 新呼叫发起
- ANDON_ESCALATE: 升级触发
- ANDON_RESPONSE: 有人响应
- ANDON_RESOLVE: 处理完成
```
