## 1. 系统概述

### 1.1 项目简介

SFMS3.0（Smart Factory Management System 3.0）是一款面向离散制造企业的智能工厂管理系统，基于MOM（Manufacturing Operations Management）理念构建，涵盖生产执行、质量管控、设备管理、供应链管理、仓库管理、流程审批等核心业务模块。

### 5.2 公共字段设计

所有业务表继承 `BaseDO`，自动包含：

```sql
CREATE TABLE xxx (
    id            BIGINT PRIMARY KEY AUTO_INCREMENT,
    create_time   DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    update_time   DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    creator       BIGINT DEFAULT 0,
    updater       BIGINT DEFAULT 0,
    deleted       TINYINT DEFAULT 0,
    tenant_id     BIGINT DEFAULT 1,
    -- 业务字段...
);
```

### 5.3 主键策略

- **本地模块**: `@KeySequence` 注解 + 数据库序列
- **跨模块ID**: 分布式ID生成器（预留）

## 6. API设计规范

### 6.1 RESTful URL规范

```
模块前缀: /{module}/{submodule}

操作映射:
┌─────────┬────────────┬──────────────────┐
│  方法    │    操作    │       示例        │
├─────────┼────────────┼──────────────────┤
│  POST   │   创建     │ POST /mes/orderday/create │
│  PUT    │   更新     │ PUT /mes/orderday/update  │
│  DELETE │   删除     │ DELETE /mes/orderday/delete │
│  GET    │   查询     │ GET /mes/orderday/get/{id} │
│  GET    │   分页查询 │ GET /mes/orderday/page     │
│  GET    │   列表查询 │ GET /mes/orderday/list     │
│  POST   │   导入     │ POST /mes/orderday/import  │
│  GET    │   导出     │ GET /mes/orderday/export   │
│  POST   │   业务操作 │ POST /mes/orderday/publish │
└─────────┴────────────┴──────────────────┘
```

### 6.2 统一响应格式

```java
{
    "code": 0,           // 状态码，0=成功
    "data": { },         // 业务数据
    "msg": "success",    // 消息
    "traceId": "xxx"     // 请求追踪ID
}

// 分页响应
{
    "code": 0,
    "data": {
        "list": [ ],
        "total": 100,
        "pageNo": 1,
        "pageSize": 20
    },
    "msg": "success"
}
```

## 10. 开发指导

### 10.1 新模块开发流程

1. **创建模块骨架**: 使用Infra代码生成器生成CRUD模板
2. **定义DO实体**: 继承BaseDO，添加业务字段
3. **实现Service层**: 业务逻辑封装
4. **实现Controller层**: RESTful API暴露
5. **配置权限**: 在System模块配置菜单和权限标识
6. **编写单元测试**: 确保核心逻辑覆盖

### 10.2 跨模块调用

- **同进程调用**: 通过`XxxApi`接口直接调用
- **异步消息**: 通过Infra定时任务或消息队列（预留）

### 10.3 配置管理

- **开发环境**: `application-dev.yml`
- **测试环境**: `application-test.yml`
- **生产环境**: `application-prod.yml`
