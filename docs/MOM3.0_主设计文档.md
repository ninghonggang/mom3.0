## 1. 系统概述

### 1.1 项目背景

MOM3.0是面向离散制造业的智能制造执行系统，覆盖从订单到交付的全流程生产管理。

### 1.2 设计目标

- **纵向贯通**: 从订单接收到生产完工的全流程追溯
- **横向协同**: 采购、仓储、质量、设备的多部门协同
- **实时可控**: 车间级实时数据采集与监控
- **决策支撑**: 生产、交付、质量的多维度数据分析

### 1.3 适用范围

- 离散制造业（装配型生产线）
- 多车间多产线协同生产模式
- 需严格质量追溯的精密制造

## 4. 技术架构

### 4.1 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | Vue3 + TypeScript + Element Plus + ECharts |
| 后端 | Go + Gin + GORM |
| 数据库 | PostgreSQL |
| 缓存 | Redis |
| 文件存储 | 本地/OSS |

### 4.2 项目结构

```
mom3.0/
├── mom-server/           # Go后端
│   ├── cmd/              # 入口
│   ├── internal/
│   │   ├── handler/      # HTTP处理层
│   │   ├── service/      # 业务逻辑层
│   │   ├── repository/    # 数据访问层
│   │   ├── model/        # 数据模型
│   │   ├── router/       # 路由配置
│   │   └── middleware/   # 中间件
│   └── data/             # SQL脚本
├── mom-web/              # Vue前端
│   └── src/
│       ├── api/           # API调用
│       ├── views/         # 页面组件
│       ├── components/    # 公共组件
│       ├── stores/       # 状态管理
│       └── router/       # 路由配置
└── docs/                # 设计文档
```

### 4.3 分层架构

```
HTTP Request
    ↓
Middleware (JWT/Auth/Log)
    ↓
Handler (参数校验/响应封装)
    ↓
Service (业务逻辑/事务)
    ↓
Repository (数据访问/ORM)
    ↓
Database (PostgreSQL)
```

## 5. 模块间关系

### 5.1 核心流程

```
销售订单 → MPS计划 → 日计划 → 工单 → 派工 → 报工 → 完工入库
```

## 6. UI设计规范

### 6.1 设计原则

- **清晰优先**: 信息层次分明，关键数据一眼可见
- **高效操作**: 常用操作3步内完成
- **一致性**: 同类场景统一交互模式
- **容错性**: 操作可撤销，错误有提示

### 6.2 色彩系统

| 用途 | 颜色 | Hex |
|------|------|-----|
| 主色 | 深蓝 | `#2563eb` |
| 成功/正常 | 绿色 | `#10b981` |
| 警告/待处理 | 橙色 | `#f59e0b` |
| 异常/不合格 | 红色 | `#ef4444` |
| 信息/进行中 | 蓝色 | `#2563eb` |
| 完成/禁用 | 灰色 | `#6b7280` |

### 6.3 页面布局

```
┌────────────────────────────────────┐
│           搜索区域 (折叠式)          │
├────────────────────────────────────┤
│  工具栏 (新增/批量操作 | 展开搜索)   │
├────────────────────────────────────┤
│                                    │
│           表格区域                   │
│                                    │
├────────────────────────────────────┤
│         分页 (右下角)               │
└────────────────────────────────────┘
```

### 6.4 组件规范

| 组件 | 规范 |
|------|------|
| 按钮-主操作 | `type="primary"`，蓝色 |
| 按钮-次操作 | `type="default"` 或无类型 |
| 按钮-危险操作 | `type="danger"`，红色，需二次确认 |
| 表格 | `stripe border v-loading` |
| 状态标签 | `el-tag` + 语义化颜色 |
| 表单校验 | `FormRules` + 实时校验 |

详细规范见 [MOM3.0_UI设计规范.md](./MOM3.0_UI设计规范.md)

## 8. API设计

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /resource/list | 列表查询 |
| GET | /resource/:id | 详情查询 |
| POST | /resource | 新增 |
| PUT | /resource/:id | 更新 |
| DELETE | /resource/:id | 删除 |
| POST | /resource/:id/action | 业务操作 |

### 8.2 响应格式

```json
{
  "code": 0,
  "msg": "success",
  "data": {},
  "meta": {
    "total": 100,
    "page": 1,
    "page_size": 20
  }
}
```

### 8.3 认证

- JWT Token认证
- Token在请求头 `Authorization: Bearer <token>`
- Token有效期7天
