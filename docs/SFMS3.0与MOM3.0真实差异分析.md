## 1. 页面重复设计分析

### 1.1 SFMS3.0重复模式

SFMS3.0存在严重的页面重复设计，主要模式：

| 重复模式 | 示例 | 说明 |
|---------|------|------|
| Job+Record+Request | `/issueJobMain` + `/issueRecordMain` + `/issueRequestMain` | 同一业务的作业/记录/请求三个页面 |
| addForm+detail+useAddForm | `/addForm` + `/detail` + `/useAddForm` | 同一个功能的多种表单 |
| 不同后缀 | `RecordMainHold` + `RecordMain` | 同一记录的不同状态版本 |
