# 编码风格规范

本目录定义 MES3.0 项目的编码风格标准。

## 文件清单

- **[go-backend.md](./go-backend.md)** - Go后端编码规范
- **[vue-frontend.md](./vue-frontend.md)** - Vue3前端编码规范
- **[ui-design.md](./ui-design.md)** - UI设计规范

## 核心原则

### 1. 可读性优先 (CRITICAL)
- 代码首先是给人读的，其次是给机器读的
- 变量/函数命名必须清晰表达意图
- 避免过于简洁的缩写（除非是公认的惯例如 `id`, `req`, `res`）

### 2. 一致性至上 (CRITICAL)
项目- 整个保持统一的代码风格
- 新代码必须遵循现有代码的风格
- 团队一致性比个人偏好更重要

### 3. 简洁为美 (HIGH)
- 避免过度工程化
- 不要为了"可能的需求"添加抽象层
- 代码量与复杂度成正比

### 4. 错误处理 (CRITICAL)
- 永远不要忽略错误
- 错误信息必须对用户友好
- 详细日志留给后端，前端展示友好提示

### 5. 安全性 (CRITICAL)
- 所有用户输入必须验证
- 禁止硬编码敏感信息
- 遵循最小权限原则
